# VR Arcane Battle

Meta Quest 환경에서 손 제스처, 헤드 트래킹, 보이스 입력을 활용해 마법을 시전하고 보스와 결투하는 VR 마법 액션 게임 프로젝트입니다.

본 프로젝트는 **VR 마법사 결투 / Arcane Duel VR** 아이디어 제안 발표를 바탕으로 시작되었습니다. 단순히 손동작으로 마법을 발사하는 것을 넘어, 속성 선언, 양손 조합, 보스 상태 카운터, 마나 관리, 회피와 베리어를 결합한 전략형 VR 전투 경험을 목표로 합니다.

## 프로젝트 개요

- 장르: VR 핸드트래킹 기반 마법 액션 / 보스 배틀
- 플랫폼: Meta Quest 2 / 3 / 3S
- 주요 입력: Hand Tracking, Head Direction, Voice Command, 6DoF Tracking
- 개발 엔진: Unity
- 핵심 목표: 컨트롤러 없이 손과 몸의 움직임만으로 몰입감 있는 마법 전투 구현

## 핵심 콘셉트

플레이어는 손 제스처로 속성을 선언하고, 공격 타입을 선택해 마법을 발동합니다. 보스는 방어, 차징, 약점 노출 같은 상태를 가지며, 플레이어는 상황에 맞는 속성 조합과 회피 방식으로 대응해야 합니다.

> 속성을 선언하고, 적 상태를 읽고, 양손으로 카운터하며 전략적으로 운용하는 VR 마법 전투

## 주요 기능

### 1. 속성 선언 + 공격 타입 시스템

플레이어는 손 제스처로 속성을 먼저 선언한 뒤, 공격 타입 제스처를 통해 마법을 발동합니다.

| 속성 | 제스처 | 효과 방향 |
| --- | --- | --- |
| 화염 | V자 모양 | 지속 피해 |
| 냉기 | OK 사인 | 이동 둔화 |
| 전기 | 뿔 모양 | 일시 경직 |

공격 타입은 다음과 같이 구분합니다.

| 공격 타입 | 제스처 | 특징 |
| --- | --- | --- |
| 총형 | 검지 앞으로 | 직선 발사 |
| 에너지파 | 손바닥 앞으로 | 넓은 범위 |
| 충전 공격 | 주먹 앞으로 | 강력한 일격 |

### 2. 양손 조합 마법

서로 다른 속성을 양손으로 조합하여 보스 상태를 카운터합니다.

| 조합 | 대응 상태 | 효과 |
| --- | --- | --- |
| 화염 + 냉기 | 방어 상태 | 방어 파괴 / 경직 |
| 냉기 + 전기 | 차징 상태 | 차징 취소 / 속박 |
| 전기 + 화염 | 약점 노출 | 고데미지 공격 |

### 3. 상태 기반 보스 전투

보스는 체력 구간과 전투 상황에 따라 상태가 변화합니다.

- 방어 상태: 일반 공격 효율 감소, 조합 마법으로 방어 파괴
- 차징 상태: 강력한 공격 준비, 특정 조합으로 차징 취소
- 약점 노출 상태: 외피 균열과 코어 노출, 고데미지 기회 제공

플레이어는 보스의 시각적 피드백을 읽고 적절한 마법 조합을 선택해야 합니다.

### 4. 회피 & 베리어 시스템

상대 공격은 단순 UI 버튼이 아니라 신체 움직임으로 대응합니다.

| 공격 유형 | 대응 방식 |
| --- | --- |
| 상단 공격 | 고개 숙이기 |
| 중단 공격 | 고개 좌우 이동 또는 베리어 |
| 하단 공격 | 베리어 필수 |

베리어는 양손 주먹 제스처로 발동하며, 마나를 소모합니다. 따라서 방어를 남발하면 공격 자원이 부족해지는 전략적 선택이 발생합니다.

### 5. Hand Pull 이동 + 속박 전투

자유 이동 상태에서는 손을 앞으로 뻗고 뒤로 당기는 방식으로 이동합니다.

전투 중 특정 보스 패턴이 발동되면 플레이어는 적정 거리로 보정된 뒤 속박 상태에 들어가며, 이때는 끌기 이동이 제한되고 회피, 베리어, 마도서 확인 중심의 대응이 요구됩니다.

### 6. 마도서 시스템

왼손 펼치기 제스처로 인게임 마도서를 소환합니다.

마도서에서는 다음 정보를 확인할 수 있습니다.

- 속성 제스처 목록
- 공격 타입 조합표
- 양손 조합 마법
- 보스 상태별 대응법
- 회피 및 베리어 사용법

마도서는 튜토리얼을 대체하는 인게임 참조 UI로, 초보자는 조작법을 확인하고 숙련자는 전투 중 전략 정보를 빠르게 확인할 수 있도록 설계합니다.

### 7. 마나 & 보이스 주문 시스템

마나는 4칸 구조로 관리합니다.

| 행동 | 마나 소모 |
| --- | --- |
| 단일 속성 마법 | 1칸 |
| 양손 조합 마법 | 2칸 |
| 베리어 | 1칸 |

보이스 입력은 마법 시전의 주 입력이 아니라 보조 보상 시스템으로 사용합니다. 제스처 시전 타이밍에 맞춰 주문을 외치면 마나 일부를 환급받습니다.

예상 주문 예시:

| 속성 | 주문 |
| --- | --- |
| 화염 | Ignis |
| 냉기 | Glacies |
| 전기 | Fulgur |

## HCI 설계 방향

이 프로젝트는 VR 환경에서 자연스러운 신체 기반 인터랙션을 만드는 것을 중요하게 다룹니다.

- External Representation: 손 오라 색상으로 현재 속성 상태 표현
- Visibility: 마도서를 통해 조작법과 대응 정보를 게임 안에서 확인
- Multimodal Input: 손 제스처와 보이스 입력을 함께 사용
- Embodied Interaction: 고개 숙이기와 기울이기로 실제 회피 수행
- Natural Mapping: 손으로 공간을 끌어 이동하는 물리적 메타포 사용
- State-based Feedback: 보스 상태를 이펙트로 표현해 대응 전략을 직관적으로 전달
